\# Car Analytics Database



\## 📋 Описание



База данных для системы аналитики автомобилей и диагностики ошибок OBD2. Разработана в рамках курсового проекта по дисциплине \*\*МДК.11.01 "Технология разработки и защиты баз данных"\*\*.



\*\*Назначение:\*\* Хранение информации об автомобилях, кодах ошибок OBD2, пользователях, избранном и сервисной истории.



\*\*СУБД:\*\* PostgreSQL 14+  

\*\*Кодировка:\*\* UTF-8  

\*\*Нормальная форма:\*\* 3NF (третья нормальная форма)



\---



\## 🏗️ Структура базы данных



\### Таблицы (8 штук)



| Таблица | Описание | Количество полей |

|---------|----------|-----------------|

| `error\_categories` | Категории ошибок OBD2 | 7 |

| `errors` | Ошибки OBD2 с описаниями и решениями | 14 |

| `cars` | Автомобили различных марок и моделей | 14 |

| `car\_errors` | Связь машина-ошибка (многие-ко-многим) | 4 |

| `users` | Пользователи системы | 11 |

| `favorites` | Избранное пользователя | 7 |

| `service\_records` | Сервисная история обслуживания | 12 |

| `notifications` | Уведомления пользователей | 8 |



\### ERD-диаграмма

\*\*Связи между таблицами:\*\*

\- `error\_categories` → `errors` (один-ко-многим)

\- `users` → `favorites` (один-ко-многим)

\- `users` → `notifications` (один-ко-многим)

\- `cars` → `service\_records` (один-ко-многим)

\## 👥 Роли пользователей



| Роль | Описание | Права доступа |

|------|----------|---------------|

| `car\_admin` | Администратор базы данных | Полный доступ ко всем таблицам, функциям и процедурам |

| `app\_user` | Обычный пользователь мобильного приложения | Чтение машин и ошибок, управление избранным и сервисной историей |

| `service\_user` | Сервисный пользователь (аналитик) | Чтение всех данных, выполнение аналитических функций |
